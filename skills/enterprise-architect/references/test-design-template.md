## Purpose
Software unit tests are used to confirm that each unit of the software code performs as expected. Unit Tests are executed by developers to isolate a section of code and verify its correctness. A unit may be an individual function, method, procedure, module, or object.

| Software unit tests are used to test the design accuracy of modules in custom developed software, and / or configuration, and are based on the Software Detailed Design Description and source code. 
This Unit Test Script template applies to testing of custom code or configured functionality under the control of Abbott BTS.
Wherever possible, configuration should be treated as source code and checked into the appropriate source code management tool. Test cases and conditions should be clearly documented to ensure acceptable results are achieved. 
Expectations for a well-written unit test script include positive tests, and boundary conditions. Software unit tests should be repeatable and under version control. Software unit tests may support test plan requirements, take various formats, and contain:
•	Software unit test name and document version
•	Application/System name
•	Revision history 
•	Setup requirements (e.g., environment, data) 
•	A uniquely identifiable test step number within each unit
•	Clear, concise, and unambiguous test cases, test conditions, and objectives
•	One pass/fail indicator per test condition
•	Test executor signature and date
•	Documentation of executed test results
