# How to Publish Documentation from Claude to Confluence

This guide explains how to publish project documentation generated by Claude directly to the Abbott Confluence space using the publish script in this repository.

---

## Prerequisites

- Access to the `DTSTest` repository on GitHub
- Python 3 installed on your machine
- An Atlassian API token (see Step 1 below)
- The `requests` Python library (`pip install requests`)

---

## Step 1: Generate an Atlassian API Token

1. Go to: https://id.atlassian.com/manage-profile/security/api-tokens
2. Click **Create API token**
3. Give it a label (e.g. `Claude Confluence Publish`)
4. Click **Create** and **copy the token** — you will not be able to see it again

> Tokens expire. If you get a `401 Unauthorized` error, generate a new token.

---

## Step 2: Clone or Navigate to the Repository

```bash
cd /path/to/DTSTest
```

If you haven't cloned it yet:

```bash
git clone git@github.com:lalitha-gundlapalli_ABT/DTSTest.git
cd DTSTest
```

---

## Step 3: Run the Publish Script

From the repository root, run:

```bash
CONFLUENCE_USER_EMAIL="your.email@abbott.com" \
CONFLUENCE_API_TOKEN="your-api-token-here" \
CONFLUENCE_BASE_URL="https://devops-abbott.atlassian.net" \
CONFLUENCE_SPACE_KEY="ABTAIGAI" \
CONFLUENCE_PARENT_FOLDER_ID="54852222981" \
python3 .github/scripts/publish_to_confluence.py
```

Replace:
- `your.email@abbott.com` with your Atlassian login email
- `your-api-token-here` with the token from Step 1

## Adding New Documents to Publish

To publish additional files, edit `.github/scripts/publish_to_confluence.py` and add an entry to the `docs` list:

```python
docs = [
    {
        "title": "My New Page Title",
        "path": repo_root / "path/to/your/file.md",
        "type": "markdown",   # use "html" for HTML files
    },
    # ... existing entries
]
```

### Supported File Types

| Type | Value | Notes |
|------|-------|-------|
| Markdown | `"markdown"` | Converts headings, bold, tables, code blocks |
| HTML | `"html"` | Extracts body content, strips scripts and styles |

---

## Publishing Under a Different Parent Page

By default, all pages are published under the **FAQ Chatbot Project** parent page.

To publish under a different parent, pass a different `parent_id` when calling `create_or_update_page` in the script. You can find a page's ID in its Confluence URL:

```
https://devops-abbott.atlassian.net/wiki/spaces/ABTAIGAI/pages/PAGE_ID/...
```

---

## Re-running the Script

The script is **idempotent** — running it again updates existing pages rather than creating duplicates. It checks for existing pages by title before creating new ones.

---

## Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `401 Unauthorized` | API token expired or incorrect | Generate a new token at id.atlassian.com |
| `SSL certificate verify failed` | Corporate proxy CA | Already handled — script uses `verify=False` |
| `403 Forbidden` | No permission to edit the space | Contact your Confluence space admin |
| `Page not found` | Wrong folder ID | Check the URL of the target folder |
| `ModuleNotFoundError: requests` | Library not installed | Run `pip install requests` |
